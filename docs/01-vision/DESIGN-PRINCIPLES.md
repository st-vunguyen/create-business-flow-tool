# Design Principles

These principles are non-negotiable. Every contribution — code, agent, rule, skill, or prompt — must honor them.

---

## 1. Evidence-first

Every claim in an output artifact must trace to a specific line in the source spec.

- No actor, step, decision, outcome, or test case may be added without a citation.
- Preferred citation format: `[source-file.md L45]`
- When evidence is missing: write `Unknown / needs confirmation`. Never guess.

---

## 2. Spec-only

The tool reads specs. It does not:

- Infer requirements from industry best practices
- Assume behaviors because they are "obvious"
- Borrow patterns from similar systems it has seen

If the spec doesn't say it, the tool doesn't output it.

---

## 3. Vendor-agnostic AI asset system

AI agents, rules, skills, and prompts live in top-level canonical directories (`agents/`, `rules/`, `skills/`, `prompts/`). These are the single source of truth.

Tool-specific formats (`.claude/`, `.github/prompts/`) are generated adapter layers, not sources of truth. They are regenerated by running `node tools/sync-adapters.mjs`.

This means the platform can serve any AI tool — Claude, GitHub Copilot, Cursor, or a future tool — without rewriting the logic.

---

## 4. No symlink dependency

Adapters are generated file copies, not symlinks. This ensures compatibility on Windows, in Docker containers, and in CI environments where symlink resolution may fail.

---

## 5. Two independent pipelines

Business-flow analysis and test-strategy analysis are independent pipelines. They share:
- The same normalized corpus (`01-source/normalized-spec.md`)
- The same evidence rules
- The same evidence-first philosophy

They do not share output directories, section schemas, or artifact formats.

---

## 6. Heuristic-first, LLM-optional

The default mode is `heuristic` — deterministic, pattern-based, no API keys required. This ensures the tool works in any environment, with any spec.

LLM enrichment (`--mode llm`) is optional and additive. It enriches heuristic output; it does not replace it.

---

## 7. Traceability is a hard quality gate

Every output document must include a traceability section that maps each output row to its source evidence. A document without traceability is not a valid artifact.

---

## 8. English only

All output artifacts, labels, headings, and comments use English only. No emoji, no slang, no non-English text in any artifact section.
